# Приклади для Bulk Import

## Структура

```
examples/
├── product-example.json      # Повний приклад JSON файлу товару
├── products-example.csv      # Приклад CSV файлу з кількома товарами
└── README.md                 # Цей файл
```

## Як використовувати

### JSON формат (рекомендовано):

1. Створіть папку для товару: `товари/lunnytsya-z-perlamy/`
2. Скопіюйте `product-example.json` в цю папку як `product.json`
3. Відредагуйте дані під ваш товар
4. Додайте зображення в ту ж папку
5. Запустіть bulk import

### CSV формат:

1. Відкрийте `products-example.csv`
2. Додайте свої товари в тому ж форматі
3. Збережіть як `products.csv`
4. Запустіть bulk import з CSV файлом

## Примітки

- JSON підтримує всі функції системи
- CSV підтримує тільки основні поля
- Для багатомовності використовуйте JSON
- Зображення завжди в окремих папках для кожного товару

## Markdown Тексти

### Варіант 1: Прямо в JSON
Пишіть Markdown прямо в полях `description_*` та `legend_content_*`:
```json
{
  "description_uk": "Текст з **жирним** та *курсивом*.\n\n## Заголовок"
}
```

### Варіант 2: Окремі .md файли
Для довгих текстів використовуйте окремі `.md` файли:
```json
{
  "description_uk_file": "description-uk.md"
}
```

**Приклади .md файлів:**
- `description-uk-example.md` - приклад опису товару
- `legend-content-uk-example.md` - приклад контенту легенди

**Переваги .md файлів:**
- ✅ Легше редагувати довгі тексти
- ✅ Підтримка Markdown редакторів
- ✅ Чистіший JSON файл
